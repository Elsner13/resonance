# FRAMEWORK Carousel Template

**Best for:** Actionable tools coaches, creators, or thinkers can apply immediately.
**Slide count:** 10 slides
**Hook pattern:** `The [N] [things] every [person] [does/asks/needs].`

---

## Slide Arc

### Slide 1 — Hook
**Role:** The numbered hook. Specific. Targeted. Implies the reader is missing something.
**Headline layer (`slide-1-headline`):** `The [N] [things] every [person] [does/asks/needs].`
**Headline-accent layer (`slide-1-headline-accent`):** The number N — DM Sans Bold, gold `#C8A96E`
**Body layer (`slide-1-body`):** Leave empty or add one half-sentence of context maximum.
**Label layer (`slide-1-label`):** `ATTUNE` (Geist Mono, tracked caps, gold)
**Tone:** Direct. The number signals completeness. The specificity signals authority.

**Copy prompt:** Write the hook as a numbered statement targeting a specific person (coach, creator, athlete, writer, parent). N is typically 3 — the minimum for a framework, the maximum for a single carousel. The number N is the `slide-1-headline-accent` content — it is the gold-accented word in the headline. Examples:
- "The **3** questions every coach should ask before designing a drill."
- "The **3** things every creator gets wrong about practice."
- "The **3** constraints every skill environment needs."

---

### Slide 2 — Stakes
**Role:** Why these [N] things matter. What happens when they're missing.
**Headline layer (`slide-2-headline`):** `Most [people] skip these. Here's why that matters.`
**Body layer (`slide-2-body`):** 2-3 sentences on the cost of not using this framework. Specific consequence — not vague.
**Tone:** Honest. Not alarmist. The stakes should feel real, not manufactured.

**Copy prompt:** What is the actual cost of not asking these questions or applying this framework? Name a specific failure mode: athletes who train hard but can't transfer skill to competition, creators who produce a lot but grow slowly, coaches who get compliance but not development. One concrete consequence per slide.

---

### Slide 3 — Principle #1 Name
**Role:** Introduce the first principle. Name it. Define it in one sentence.
**Headline layer (`slide-3-headline`):** `[#1]: [Principle Name]`
**Headline-accent layer (`slide-3-headline-accent`):** The principle name — DM Sans Bold, gold `#C8A96E`
**Body layer (`slide-3-body`):** One sentence definition. Plain language. No jargon without translation.
**Label layer (`slide-3-label`):** `01 / 03` (Geist Mono, tracks where we are in the framework, gold)
**Tone:** Clear. Assertive. This is a named principle, not a tip.

**Copy prompt:** Name the first principle with a short, memorable title (2-4 words max). Then define it in one sentence a practitioner can act on immediately. Examples: "Representativeness: The practice environment must contain the same information that the performance environment demands." / "Affordance Design: The constraint is the instruction."

---

### Slide 4 — Principle #1 in Practice (left-border list)
**Role:** Concrete example of Principle #1 applied. Use left-border accent list layout.
**Headline layer (`slide-4-headline`):** `In practice:`
**List-1-title layer (`slide-4-list-1-title`):** First application point title (Geist Mono, all caps, cream bold)
**List-1-body layer (`slide-4-list-1-body`):** First application point description (DM Sans, cream 70% opacity)
**List-2-title layer (`slide-4-list-2-title`):** Second application point title
**List-2-body layer (`slide-4-list-2-body`):** Second application point description
**List-3-title layer (`slide-4-list-3-title`):** Third application point title (optional)
**List-3-body layer (`slide-4-list-3-body`):** Third application point description (optional)
**Tone:** Practical. The reader should be able to do this within their next session, post, or conversation.

**Copy prompt:** Write 2-3 specific application points using the left-border list format (like BJJMM's "REGULAR COMPETITION / TEAM CULTURE / A REASON TO TRAIN" slide). Each point: a bold title + 1-2 sentence description. Domain-specific to the target reader but crossing into a second domain if it deepens the example.

---

### Slide 5 — Principle #2 Name
**Role:** Second principle. Different domain example than #1 if possible.
**Headline layer (`slide-5-headline`):** `[#2]: [Principle Name]`
**Headline-accent layer (`slide-5-headline-accent`):** The principle name — DM Sans Bold, gold `#C8A96E`
**Body layer (`slide-5-body`):** One sentence definition.
**Label layer (`slide-5-label`):** `02 / 03` (Geist Mono, gold)
**Tone:** Same as Slide 3 — named, assertive, clear.

**Copy prompt:** Name and define the second principle. It should be distinct from but complementary to #1. Together they should feel like a coherent system, not a random list. If #1 was about the environment, #2 might be about the practitioner's orientation or the feedback structure.

---

### Slide 6 — Principle #2 in Practice (left-border list)
**Role:** Concrete example of Principle #2. Different domain from slide 4 if possible.
**Headline layer (`slide-6-headline`):** `In practice:`
**List-1-title layer (`slide-6-list-1-title`):** First application point title
**List-1-body layer (`slide-6-list-1-body`):** First application point description
**List-2-title layer (`slide-6-list-2-title`):** Second application point title
**List-2-body layer (`slide-6-list-2-body`):** Second application point description
**List-3-title layer (`slide-6-list-3-title`):** Third point title (optional)
**List-3-body layer (`slide-6-list-3-body`):** Third point description (optional)
**Tone:** Same as Slide 4 — practical, vivid.

**Copy prompt:** Write the practical example for Principle #2. If slide 4 used a sport context, use creation, writing, or parenting here. The cross-domain application shows the framework is a lens, not a sport-specific tool.

---

### Slide 7 — Principle #3 Name
**Role:** Third principle. The framework feels complete after this.
**Headline layer (`slide-7-headline`):** `[#3]: [Principle Name]`
**Headline-accent layer (`slide-7-headline-accent`):** The principle name — DM Sans Bold, gold `#C8A96E`
**Body layer (`slide-7-body`):** One sentence definition.
**Label layer (`slide-7-label`):** `03 / 03` (Geist Mono, gold)
**Tone:** Same as slides 3 and 5. Often the most counterintuitive of the three.

**Copy prompt:** Name and define the third principle. It should feel like the completion of the set — the thing that makes the framework whole. Often this is the most counterintuitive of the three, or the one that reframes how to use #1 and #2.

---

### Slide 8 — Principle #3 in Practice (left-border list)
**Role:** Concrete example of Principle #3.
**Headline layer (`slide-8-headline`):** `In practice:`
**List-1-title layer (`slide-8-list-1-title`):** First application point title
**List-1-body layer (`slide-8-list-1-body`):** First application point description
**List-2-title layer (`slide-8-list-2-title`):** Second application point title
**List-2-body layer (`slide-8-list-2-body`):** Second application point description
**List-3-title layer (`slide-8-list-3-title`):** Third point title (optional)
**List-3-body layer (`slide-8-list-3-body`):** Third point description (optional)
**Tone:** Same as slides 4 and 6.

**Copy prompt:** Write the practical example for Principle #3. By this point, the reader has seen the framework in two different domains. This third example should either surprise them (unexpected domain) or bring the framework home to the domain most prominent in slide 1.

---

### Slide 9 — Synthesis
**Role:** One sentence that ties the entire framework together. The philosophical payload.
**Headline layer (`slide-9-headline`):** `The through-line:`
**Body layer (`slide-9-body`):** One sentence only. The synthesis statement. The sentence someone screenshots.
**Accent layer (`slide-9-accent`):** The synthesis sentence repeated in larger Cormorant Garamond Italic, gold — for visual weight.
**Tone:** Distilled. Aphoristic. This should feel like it took years to arrive at.

**Copy prompt:** Write one sentence that synthesizes all three principles into a single insight. This is not a summary — it is a compression. The three principles should disappear into this sentence like variables resolved into an answer. It should be quotable on its own, out of context. Examples: "The environment is not the backdrop for skill. It is the source." / "You cannot prescribe perception. You can only design the conditions for it to emerge."

---

### Slide 10 — CTA
**Role:** Close. Save ask. One conversion path.
**Headline layer (`slide-10-headline`):** `Save this framework. Link in bio for Signal/Noise.`
**Body layer (`slide-10-body`):** Optional: "New frameworks every week. Free."
**Label layer (`slide-10-label`):** `@attunemastery` (Geist Mono, gold)
**Tone:** Generous. The framework was free. Signal/Noise is also free. The relationship builds.

**Copy prompt:** Write a save CTA (algorithmic necessity) and one conversion CTA pointing to Signal/Noise. Frameworks are the highest-save content format on Instagram — lean into this. The CTA should feel like a natural continuation of the value already given, not a pivot to selling.

## Layout Modules Used in FRAMEWORK

**Slide 1:** Full-bleed headline. Number in gold DM Sans Bold. Brand mark.
**Slide 2:** Headline + body. Stakes slide — spare and direct.
**Slides 3, 5, 7:** Headline (principle name in gold) + one-sentence body + progress label (`01/03` etc.)
**Slides 4, 6, 8:** Headline + left-border accent list (BJJMM style — 2-3 items with title + description, gold left border, 4px width)
**Slide 9:** Headline (`The through-line:`) + one sentence body + accent (same sentence, larger, gold Cormorant)
**Slide 10:** Headline + optional 1-line body + label = @attunemastery

## Voice Checklist (run before finalizing all slides)

- [ ] First person (I) throughout
- [ ] No em dashes — colon or period
- [ ] No banned words: delve, realm, harness, unlock, tapestry, leverage, synergy, seamless, optimize
- [ ] No AI transitions
- [ ] Slides 3, 5, 7 have consistent structure: `[#N]: [Name]` + one-sentence definition
- [ ] Slides 4, 6, 8 use different domains across the three examples
- [ ] Slide 9 is one sentence only — no elaboration
- [ ] At least two domains crossed total
- [ ] Framework feels complete — not like a list, like a system
